How signing works

1
Report is generated
integrity.molt scans the address using multi-agent AI swarm. The report text is produced as a structured analysis.
2
Ed25519 signature is applied
The report text is signed using integrity.molt's Ed25519 private key (stored in HSM-equivalent secure storage). The signature is deterministic and covers the exact report bytes.
3
Verify key is published
The corresponding public verify key is published at /services and /.well-known/x402.json. Anyone can verify any report against the public key.
4
key_id enables key rollover
Each report contains key_id (first 16 chars of the verify key in base64). This allows verifying older reports even after key rotation.

Verify with Python

pip install PyNaCl python3 verify-report.py report.signed.json # Or inline: python3 - <<'EOF' import json, base64, nacl.signing with open('report.signed.json') as f: envelope = json.load(f) verify_key_bytes = base64.b64decode(envelope['verify_key']) signature_bytes = base64.b64decode(envelope['signature']) report_bytes = envelope['report'].encode('utf-8') vk = nacl.signing.VerifyKey(verify_key_bytes) try: vk.verify(report_bytes, signature_bytes) print("✓ VALID — report is authentic and untampered") except Exception as e: print(f"✗ INVALID — {e}") EOF
