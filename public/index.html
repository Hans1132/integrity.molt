Oracle vs Scanner

Why not just a scanner?

Most security tools return human-readable output. integrity.molt returns composable signed envelopes.

Composable receipts

A signed envelope is portable JSON. An agent can forward it, chain it into a decision, cache it, or hand it to an auditor — without re-fetching.

Server-verifiable signatures

valid:true requires both correct Ed25519 math AND the key matching the oracle's JWKS. Foreign-signed envelopes are explicitly flagged as key_not_pinned.

A2A JSON-first API

Every response is structured JSON — no PDF, no dashboard, no scraping required. Discoverable via /.well-known/agent-card.json.

x402 pay-per-call

No subscription, no API key negotiation. Send USDC on Solana, include the tx sig as X-Payment header. Anti-replay enforced atomically.

Offline Python verify — no HTTP call:
import json, base64, nacl.signing
receipt = json.load(open('receipt.json'))
vk = nacl.signing.VerifyKey(base64.b64decode(receipt['verify_key']))
payload = {k: v for k, v in receipt.items()
           if k not in {'signature','verify_key','key_id','signed_at','signer','algorithm','report'}}
canonical = json.dumps(payload, sort_keys=True, separators=(',', ':'))
vk.verify(canonical.encode(), base64.b64decode(receipt['signature']))
print("✓ Valid")
