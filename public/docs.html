Developer documentation

RESTful API + x402 micropayments. Production-ready for AI agents, bots, and protocol integrations.

Base URL: https://intmolt.org
Protocol: x402 (HTTP 402 + X-PAYMENT header) or Bearer API key
Payment asset: USDC on Solana mainnet (EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v)

Authentication

Two modes — choose based on your use case:

x402 micropayments (no account)

Send a base64-encoded JSON envelope in the X-PAYMENT header containing the Solana transaction signature. No registration, no API key.

Bearer API key (subscription)

Subscribers get an im_-prefixed API key. Pass it as Authorization: Bearer im_xxx to skip per-request payment.

Authorization: Bearer im_a1b2c3d4e5f6...

x402 payments

integrity.molt implements the x402 protocol. Without a payment header, endpoints return HTTP 402 with an accepts array describing what payment is required.

# 1. Probe — get payment requirements
curl -s -X POST https://intmolt.org/scan/quick \
  -H "Content-Type: application/json" \
  -d '{"address":"EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v"}'
# → HTTP 402 with accepts array

# 2. Send USDC on-chain (via Solana CLI / wallet / SDK)
# Get transaction signature, e.g.: 4kVs7...abc

# 3. Build payment envelope and scan
ENVELOPE=$(echo '{"transaction":"4kVs7...abc"}' | base64 -w0)
curl -s -X POST https://intmolt.org/scan/quick \
  -H "Content-Type: application/json" \
  -H "X-PAYMENT: $ENVELOPE" \
  -d '{"address":"EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v"}'

async function securityCheck(address: string) {
  // 1. Probe for payment requirements
  const probe = await fetch(`${INTMOLT}/scan/quick`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ address }),
  });
  const { accepts } = await probe.json();
  const req = accepts[0];

  // 2. Send USDC and get signature (your wallet/SDK)
  const txSig = await sendUsdc(req.payTo, Number(req.maxAmountRequired));

  // 3. Scan with payment envelope
  const envelope = btoa(JSON.stringify({ transaction: txSig }));
  const res = await fetch(`${INTMOLT}/scan/quick`, {
    method: "POST",
    headers: { "Content-Type": "application/json", "X-PAYMENT": envelope },
    body: JSON.stringify({ address }),
  });
  const data = await res.json();

  // 4. Verify Ed25519 signature
  const vkBytes  = Uint8Array.from(atob(data.signed.verify_key), c => c.charCodeAt(0));
  const sigBytes = Uint8Array.from(atob(data.signed.signature),   c => c.charCodeAt(0));
  const msgBytes = new TextEncoder().encode(data.signed.report);
  const valid    = nacl.sign.detached.verify(msgBytes, sigBytes, vkBytes);
  console.log(valid ? "✓ Valid report" : "✗ Invalid signature");

  return data;
}

Signed reports

Every scan response includes a signed field — an Ed25519-signed envelope you can cryptographically verify. Reports become tamper-proof attestations that AI agents can trustlessly consume.

{
  "report":     "=== integrity_molt Scan ===\n...",
  "signature":  "base64-encoded-ed25519-signature",
  "verify_key": "base64-encoded-public-key",
  "key_id":     "first-16-chars-of-key-in-base64",
  "signed_at":  "2026-03-29T12:00:00Z"
}

Verify online at /verify.html or with Python (pip install PyNaCl). The current verify key is published at /services.

Deep Audit (Swarm)

POST /scan/deep 5.00 USDC

Full multi-agent swarm audit. Four AI agents run in parallel (scanner → analyst → reputation → meta-scorecard), cross-validate findings, and produce a consensus-based signed report. Allow ~60–120 seconds.

ENVELOPE=$(echo '{"transaction":"YOUR_TX_SIG"}' | base64 -w0)
curl -s -X POST https://intmolt.org/scan/deep \
  -H "Content-Type: application/json" \
  -H "X-PAYMENT: $ENVELOPE" \
  -d '{"address":"TARGET_ADDRESS"}' | jq '{decision,aggregate_score,report}'

API keys

  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","label":"ci-pipeline"}'
